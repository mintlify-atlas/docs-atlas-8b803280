MCRIT is an open-source project welcoming contributions from the community. This guide covers how to contribute effectively to the project.

## Project Overview

**MCRIT** (MinHash-based Code Relationship & Investigation Toolkit) is a framework for applying MinHash algorithm to code similarity analysis.

- **Repository**: https://github.com/danielplohmann/mcrit
- **License**: GNU General Public License v3 (GPLv3)
- **Language**: Python 3
- **Maintainer**: Daniel Plohmann (@danielplohmann)

## Getting Started

### Prerequisites

- Python 3.8 or higher
- MongoDB 5.0+ (for storage backend testing)
- Git
- Familiarity with:
  - MinHash/LSH algorithms
  - Binary analysis and disassembly
  - REST APIs and async job processing

### Development Setup

1. **Fork and clone the repository**:

```bash
# Fork on GitHub, then clone your fork
git clone https://github.com/YOUR_USERNAME/mcrit.git
cd mcrit

# Add upstream remote
git remote add upstream https://github.com/danielplohmann/mcrit.git
```

2. **Create a virtual environment**:

```bash
# Create venv
python3 -m venv venv

# Activate
source venv/bin/activate  # Linux/Mac
venv\Scripts\activate     # Windows
```

3. **Install dependencies**:

```bash
# Install in editable mode with dev dependencies
pip install -e .
pip install -r requirements.txt

# Install testing dependencies
pip install pytest pytest-cov black flake8
```

4. **Set up MongoDB** (if testing storage):

```bash
# Install MongoDB 5.0
# See: https://docs.mongodb.com/manual/installation/

# Start MongoDB
sudo systemctl start mongod

# Verify connection
mongo --eval 'db.runCommand({ connectionStatus: 1 })'
```

5. **Verify installation**:

```bash
# Run MCRIT server
mcrit server &

# Run MCRIT worker
mcrit worker &

# Check status
mcrit client status
```

## Contributing Code

### Branching Strategy

MCRIT uses a simple branching model:

- **main**: Stable releases
- **develop**: Active development (if exists)
- **feature/**: Feature branches
- **bugfix/**: Bug fix branches

### Creating a Feature Branch

```bash
# Update main
git checkout main
git pull upstream main

# Create feature branch
git checkout -b feature/my-new-feature

# Make changes, commit frequently
git add .
git commit -m "Add my new feature"

# Push to your fork
git push origin feature/my-new-feature
```

### Code Style

MCRIT follows PEP 8 with some project-specific conventions:

#### Formatting

```bash
# Format with black
black mcrit/

# Check style
flake8 mcrit/
```

#### Naming Conventions

- **Classes**: `PascalCase` (e.g., `MinHashIndex`, `StorageInterface`)
- **Functions/Methods**: `camelCase` (e.g., `getSampleById`, `addMinHash`)
- **Variables**: `snake_case` for locals, `camelCase` for parameters
- **Constants**: `UPPER_SNAKE_CASE` (e.g., `MINHASH_SIGNATURE_LENGTH`)
- **Private methods**: `_leadingUnderscore` (e.g., `_generateByteSequences`)

### Testing

All new features should include tests:

#### Running Tests

```bash
# Run all tests
pytest

# Run with coverage
pytest --cov=mcrit --cov-report=html

# Run specific test file
pytest tests/test_storage.py

# Run specific test
pytest tests/test_storage.py::test_add_sample
```

#### Writing Tests

Create test files in `tests/` directory:

```python
# tests/test_my_feature.py
import pytest
from mcrit.config.McritConfig import McritConfig
from mcrit.shinglers.MyNewShingler import MyNewShingler

@pytest.fixture
def shingler():
    """Create shingler instance for testing."""
    config = McritConfig()
    return MyNewShingler(config.SHINGLER_CONFIG, weight=1)

def test_shingler_initialization(shingler):
    """Test shingler initializes correctly."""
    assert shingler is not None
    assert shingler._weight == 1

def test_generate_sequences(shingler):
    """Test sequence generation."""
    # Create mock function
    from tests.mock_data import create_mock_function
    function = create_mock_function()
    
    # Generate sequences
    sequences = shingler._generateByteSequences(function)
    
    # Assertions
    assert isinstance(sequences, list)
    assert len(sequences) > 0
    assert all(isinstance(s, (str, bytes)) for s in sequences)
```

### Submitting Pull Requests

1. **Ensure tests pass**:

```bash
pytest
flake8 mcrit/
```

2. **Update documentation** if needed:
   - Add docstrings to new code
   - Update README if adding features
   - Add examples if appropriate

3. **Commit with descriptive messages**:

```bash
git commit -m "Add custom shingler for control flow features

- Implement ControlFlowShingler extending AbstractShingler
- Extract loop/branch statistics as features
- Add tests for sequence generation
- Update ShinglerConfig to include new shingler"
```

4. **Push to your fork**:

```bash
git push origin feature/my-new-feature
```

5. **Create Pull Request** on GitHub:
   - Navigate to https://github.com/danielplohmann/mcrit
   - Click "Pull Requests" → "New Pull Request"
   - Select your fork and branch
   - Fill in description:
     - What does this PR do?
     - Why is it needed?
     - How was it tested?
     - Any breaking changes?

6. **Respond to review feedback**:
   - Address reviewer comments
   - Make requested changes
   - Push updates (they'll appear in PR automatically)

2. **Verify version**: Use latest MCRIT and SMDA

## Code Review Checklist

Before submitting PR, verify:

- [ ] Code follows project style conventions
- [ ] All tests pass (`pytest`)
- [ ] New code has tests (aim for >80% coverage)
- [ ] Public APIs have docstrings
- [ ] No unnecessary dependencies added
- [ ] Backwards compatibility maintained (or documented)
- [ ] Configuration changes documented
- [ ] Example usage provided (if applicable)
- [ ] Updated CHANGELOG (for significant changes)

## Release Process

Releases are managed by project maintainers:

1. Version number updated in `setup.py`
2. CHANGELOG updated in README
3. Git tag created: `git tag v1.x.x`
4. Published to PyPI: `python setup.py sdist upload`
