---
title: "Retrieve Family"
description: "Get detailed information about a specific family by ID"
---

## Endpoint

```
GET /families/{family_id}
```

Retrieve detailed information about a specific family, including optionally all samples that belong to it.

## Path Parameters

<ParamField path="family_id" type="integer" required>
  The unique identifier of the family to retrieve.
</ParamField>

## Query Parameters

<ParamField query="with_samples" type="boolean" default="true">
  Whether to include detailed sample information in the response. Set to `"false"` to exclude samples and only return family metadata.
</ParamField>

## Response

### Success Response (200 OK)

<ResponseField name="status" type="string" required>
  Status of the request. Returns `"successful"` on success.
</ResponseField>

<ResponseField name="data" type="object" required>
  The family entry object.
  
  <Expandable title="Family Entry Object">
    <ResponseField name="family_id" type="integer" required>
      Unique identifier for the family.
    </ResponseField>
    
    <ResponseField name="family_name" type="string" required>
      Name of the family.
    </ResponseField>
    
    <ResponseField name="num_samples" type="integer" required>
      Total number of samples in this family.
    </ResponseField>
    
    <ResponseField name="num_functions" type="integer" required>
      Total number of functions across all samples in this family.
    </ResponseField>
    
    <ResponseField name="num_library_samples" type="integer" required>
      Number of samples in this family that are marked as libraries.
    </ResponseField>
    
    <ResponseField name="samples" type="object">
      Dictionary of sample entries keyed by sample ID. Only included when `with_samples=true`.
      
      <Expandable title="Sample Entry Object">
        <ResponseField name="sample_id" type="integer" required>
          Unique identifier for the sample.
        </ResponseField>
        
        <ResponseField name="family_id" type="integer" required>
          The family ID this sample belongs to.
        </ResponseField>
        
        <ResponseField name="family" type="string" required>
          The family name this sample belongs to.
        </ResponseField>
        
        <ResponseField name="filename" type="string" required>
          The original filename of the sample.
        </ResponseField>
        
        <ResponseField name="sha256" type="string" required>
          SHA256 hash of the sample binary.
        </ResponseField>
        
        <ResponseField name="architecture" type="string" required>
          CPU architecture (e.g., "x86", "ARM").
        </ResponseField>
        
        <ResponseField name="bitness" type="integer" required>
          Architecture bitness (e.g., 32, 64).
        </ResponseField>
        
        <ResponseField name="base_addr" type="integer" required>
          Base address of the binary.
        </ResponseField>
        
        <ResponseField name="binary_size" type="integer" required>
          Size of the binary in bytes.
        </ResponseField>
        
        <ResponseField name="binweight" type="number" required>
          Binary weight metric.
        </ResponseField>
        
        <ResponseField name="is_library" type="boolean" required>
          Whether this sample is identified as a library.
        </ResponseField>
        
        <ResponseField name="component" type="string" required>
          Component identifier.
        </ResponseField>
        
        <ResponseField name="version" type="string" required>
          Version string of the sample.
        </ResponseField>
        
        <ResponseField name="smda_version" type="string" required>
          Version of SMDA used to analyze this sample.
        </ResponseField>
        
        <ResponseField name="timestamp" type="string" required>
          Timestamp when the sample was added (format: YYYY-MM-DDTHH-MM-SS).
        </ResponseField>
        
        <ResponseField name="statistics" type="object" required>
          Statistical information about the sample.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

### Error Response (404 Not Found)

<ResponseField name="status" type="string" required>
  Status of the request. Returns `"failed"` on error.
</ResponseField>

<ResponseField name="data" type="object" required>
  Error details.
  
  <Expandable title="Error Object">
    <ResponseField name="message" type="string" required>
      Error message: "We don't have a family with that id."
    </ResponseField>
  </Expandable>
</ResponseField>

### Retrieve family without samples

```bash
curl -X GET "http://localhost:8000/families/1?with_samples=false"
```

<Expandable title="Response">
```json
{
  "status": "successful",
  "data": {
    "family_id": 1,
    "family_name": "malware_family_a",
    "num_samples": 2,
    "num_functions": 150,
    "num_library_samples": 0
```
