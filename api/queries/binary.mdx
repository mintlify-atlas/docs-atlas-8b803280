## Query Unmapped Binary

Analyzes a binary file and finds matches in the database without storing it. The binary is automatically disassembled and analyzed.

### Request

<ParamField query="minhash_score" type="integer" default="50">
  Minimum MinHash similarity score threshold (0-100)
</ParamField>

<ParamField query="band_matches_required" type="integer" default="0">
  Minimum number of band matches required
</ParamField>

<ParamField query="pichash_size" type="integer" default="8">
  Size of PicHash for matching (must be >= 0)
</ParamField>

<ParamField query="force_recalculation" type="boolean" default="false">
  Force recalculation of matches instead of using cached results
</ParamField>

<ParamField query="sample_group_only" type="boolean" default="false">
  Only match against samples in the same group
</ParamField>

```bash
curl -X POST "https://your-server.com/query/binary?minhash_score=70&band_matches_required=2" \
  -H "apitoken: YOUR_API_TOKEN" \
  -H "Content-Type: application/octet-stream" \
  --data-binary @/path/to/binary.exe
```

```json
{
  "status": "failed",
  "data": {
    "message": "POST request without body can't be processed."
```

Returns HTTP 400 if the request has no body or the binary cannot be processed.

## Query Mapped Binary

Analyzes a binary file with a specific base address. Useful for memory dumps or shellcode analysis.

<ParamField path="base_address" type="string" required>
  Base address for the binary (hexadecimal string, e.g., "0x400000" or "400000"). Use "0" to auto-detect.
</ParamField>

<ParamField query="minhash_score" type="integer" default="50">
  Minimum MinHash similarity score threshold (0-100)
</ParamField>

<ParamField query="band_matches_required" type="integer" default="0">
  Minimum number of band matches required
</ParamField>

<ParamField query="pichash_size" type="integer" default="8">
  Size of PicHash for matching (must be >= 0)
</ParamField>

<ParamField query="force_recalculation" type="boolean" default="false">
  Force recalculation of matches instead of using cached results
</ParamField>

### Request

```bash
curl -X POST "https://your-server.com/query/binary/mapped/0x400000?minhash_score=70" \
  -H "apitoken: YOUR_API_TOKEN" \
  -H "Content-Type: application/octet-stream" \
  --data-binary @/path/to/memory_dump.bin
```

### Auto-detect Base Address

To automatically detect the base address, use "0":

```bash
curl -X POST "https://your-server.com/query/binary/mapped/0" \
  -H "apitoken: YOUR_API_TOKEN" \
  -H "Content-Type: application/octet-stream" \
  --data-binary @/path/to/binary.bin
```

### Response

The response structure is identical to the unmapped binary query endpoint.

### Use Cases

- Analyzing memory dumps with known load addresses
- Matching position-independent code (PIC) or shellcode
- Analyzing binaries extracted from memory at runtime
- Processing firmware images with specific memory layouts
