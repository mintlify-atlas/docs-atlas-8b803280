## Get Sample Matches

<ParamField path="sample_id" type="integer" required>
  The ID of the sample to match
</ParamField>

<ParamField query="minhash_score" type="integer" default="50">
  Minimum MinHash similarity score threshold (0-100)
</ParamField>

<ParamField query="band_matches_required" type="integer" default="0">
  Minimum number of band matches required
</ParamField>

<ParamField query="pichash_size" type="integer" default="8">
  Size of PicHash for matching (must be >= 0)
</ParamField>

<ParamField query="force_recalculation" type="boolean" default="false">
  Force recalculation of matches instead of using cached results
</ParamField>

<ParamField query="sample_group_only" type="boolean" default="false">
  Only match against samples in the same group
</ParamField>

### Request

```bash
curl -X GET "https://your-server.com/matches/sample/123?minhash_score=70&band_matches_required=2" \
  -H "apitoken: YOUR_API_TOKEN"
```

### Error Response

```json
{
  "status": "failed",
  "data": {
    "message": "We don't have a sample with that id."
  }
}
```

Returns HTTP 404 if the sample ID is not found in the database.

## Get Sample vs Sample Matches

<ParamField path="sample_id" type="integer" required>
  The ID of the first sample
</ParamField>

<ParamField path="sample_id_b" type="integer" required>
  The ID of the second sample to compare against
</ParamField>

<ParamField query="minhash_score" type="integer" default="50">
  Minimum MinHash similarity score threshold (0-100)
</ParamField>

<ParamField query="band_matches_required" type="integer" default="0">
  Minimum number of band matches required
</ParamField>

<ParamField query="pichash_size" type="integer" default="8">
  Size of PicHash for matching (must be >= 0)
</ParamField>

<ParamField query="force_recalculation" type="boolean" default="false">
  Force recalculation of matches instead of using cached results
</ParamField>

### Request

```bash
curl -X GET "https://your-server.com/matches/sample/123/456?minhash_score=70" \
  -H "apitoken: YOUR_API_TOKEN"
```

### Response

The response structure is identical to the sample match endpoint, containing matching information between the two specified samples.

### Error Response

```json
{
  "status": "failed",
  "data": {
    "message": "We don't have a sample with that id."
  }
}
```

Returns HTTP 404 if either sample ID is not found in the database.
